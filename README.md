## Overview

This project serves as an introduction to modern Python tooling using `uv` — a fast, all-in-one Python package and project manager. It includes a simple utility function and a fully configured pytest test suite to demonstrate best practices for project setup, dependency management, and unit testing without needing to manually activate a virtual environment.

---

## Project Structure

```
uv_pytest_intro/
├── main.py               # Application entry point and utility functions
├── tests/
│   ├── __init__.py       # Makes tests a Python package
│   └── test_main.py      # Unit tests for main.py
├── pyproject.toml        # Project metadata, dependencies, and tool config
├── uv.lock               # Locked dependency versions (auto-generated by uv)
├── .python-version       # Python version pin for this project
└── README.md             # This file
```

## Prerequisites

- **Python 3.10+**
- **uv** — install via:

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
```

Or with Homebrew:

```bash
brew install uv
```

No need to install `pytest` or create a virtual environment manually — `uv` handles both.

## Getting Started

### 1. Clone the repository

```bash
git clone <your-repo-url>
cd uv_pytest_intro
```

### 2. Sync dependencies

```bash
uv sync
```

This command reads `pyproject.toml` and `uv.lock`, creates a `.venv` virtual environment if one doesn't exist, and installs all dependencies into it. You do **not** need to activate the virtual environment manually.

## Project Source

### `main.py`

```python
def is_even(n: int) -> bool:
    return n % 2 == 0


def main():
    print("Hello from uv-pytest-intro!")


if __name__ == "__main__":
    main()
```

#### `is_even(n: int) -> bool`

Returns `True` if the given integer is even, `False` otherwise.

| Argument | Type  | Description            |
|----------|-------|------------------------|
| `n`      | `int` | The integer to evaluate |

**Examples:**

```python
is_even(4)   # True
is_even(3)   # False
is_even(0)   # True
is_even(-2)  # True
is_even(-7)  # False
```

## Testing

Tests are written using `pytest` and live in the `tests/` directory.

### Run all tests

```bash
uv run pytest
```

### Run with verbose output

```bash
uv run pytest -v
```

### Run a specific test file

```bash
uv run pytest tests/test_main.py
```

### Run tests matching a keyword

```bash
uv run pytest -k "even"
```

### Stop on first failure

```bash
uv run pytest -x
```

### How pytest discovers tests

pytest is configured in `pyproject.toml`:

```toml
[tool.pytest.ini_options]
testpaths = ["tests"]
```

This tells pytest to look inside the `tests/` directory. From there it automatically collects any file matching `test_*.py` and runs any function whose name starts with `test_`.

## Dependency Management

Dependencies are declared in `pyproject.toml`:

```toml
[project]
requires-python = ">=3.10"
dependencies = [
    "pytest>=9.0.3",
]
```

### Add a new dependency

```bash
uv add <package-name>
```

### Remove a dependency

```bash
uv remove <package-name>
```

### Update all dependencies

```bash
uv lock --upgrade
uv sync
```

The `uv.lock` file pins exact versions of all dependencies (including transitive ones) and should be committed to version control to ensure reproducible installs across machines.

## Why uv?

| Feature | Traditional workflow | With uv |
|---|---|---|
| Create virtual environment | `python -m venv .venv` | Automatic on `uv sync` |
| Activate virtual environment | `source .venv/bin/activate` | Not needed — use `uv run` |
| Install dependencies | `pip install -r requirements.txt` | `uv sync` |
| Run a script | `python main.py` | `uv run main.py` |
| Run tests | `pytest` | `uv run pytest` |
| Lock dependencies | Manual (`pip freeze`) | Automatic (`uv.lock`) |

`uv` is significantly faster than `pip` and replaces the need for separate tools like `virtualenv`, `pip-tools`, and `pyenv` for most workflows.
